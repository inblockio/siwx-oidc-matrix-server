# siwx-oidc-matrix-server

Docker Compose deployment stack that runs a Synapse Matrix homeserver fronted by
siwx-oidc (CAIP-122 OIDC provider) so agents and wallets can authenticate with
EIP-191, Ed25519, or P-256 keys.

## Repository structure

```
siwx-oidc-matrix-server/
  start-matrix.sh          # CLI entry point — parses args, writes .env, calls docker compose up
  docker-compose.yml       # Production stack (Synapse + siwx-oidc + nginx-proxy + acme-companion + Redis)
  dockerfiles/
    Dockerfile             # FROM matrixdotorg/synapse; installs yq; COPYs entrypoints/matrix_server.sh
  entrypoints/
    matrix_server.sh       # Container entrypoint — first-boot only: generates homeserver.yaml via yq
  siwx-oidc-config/
    siwe-oidc.toml         # Persistent EC P-256 signing key (generated by start-matrix.sh on first run)
  .env                     # Generated by start-matrix.sh; not committed
  .env-template            # Reference template for .env format
  .env.example             # Minimal example
```

## Services

| Service          | Image / Build                              | Port(s)       | Notes |
|------------------|--------------------------------------------|---------------|-------|
| `matrix_synapse` | `./dockerfiles/Dockerfile` (Synapse-based) | 8448          | Depends on `siwx-oidc` healthcheck |
| `siwx-oidc`      | `ghcr.io/inblockio/siwx-oidc:latest`       | `SIWEOIDC_PORT` (default 8081) | CAIP-122 OIDC provider |
| `redis`          | `redis`                                    | internal      | Session store for siwx-oidc |
| `proxy`          | `nginxproxy/nginx-proxy:alpine`            | 80, 443       | Reverse proxy; aliases both hostnames on the Docker network |
| `letsencrypt`    | `nginxproxy/acme-companion`                | —             | Auto-provisions TLS for proxy |

Volumes: `matrix_data` (Synapse data), `proxy_data_*` (nginx/acme state).

## First-boot vs. restart behaviour

`entrypoints/matrix_server.sh` only runs the `yq` configuration block when
`/data/homeserver.yaml` does not yet exist. **After first boot, changes to that
script do NOT propagate to running deployments.** To apply changes on an existing
deployment you must also edit `/data/homeserver.yaml` directly inside the
`matrix_data` volume.

## Key configuration facts

- **Synapse OIDC auth method**: `client_auth_method: client_secret_post` — siwx-oidc
  rejects `Authorization: Basic` headers, so `token_endpoint_auth_method` (the
  upstream Synapse key name) must **not** be used. Synapse's actual config key is
  `client_auth_method`.
- **`localpart_template`**: `{{ user.preferred_username | replace(':', '-') }}`
  maps `did:pkh:…` subjects to valid Matrix localparts.
- **OIDC issuer env var**: `SIWEOIDC_BASE_URL` (env prefix kept as `SIWEOIDC_`
  for backward compat even though the crate is now `siwx_oidc`).
- **Signing key**: persisted in `siwx-oidc-config/siwe-oidc.toml` (EC P-256);
  generated once by `start-matrix.sh`. Do not delete — tokens become invalid.

## Common operations

```bash
# First-time start (production)
./start-matrix.sh \
  --MATRIX_HOST matrix.example.com \
  --SIWEOIDC_HOST siwx-oidc.example.com \
  --LETSENCRYPT_EMAIL you@example.com

# Stop
./start-matrix.sh --stop

# Destroy all data (irreversible)
./start-matrix.sh --reset

# Debug mode (foreground, verbose logs)
./start-matrix.sh --ENABLE_DEBUG \
  --MATRIX_HOST matrix.example.com \
  --SIWEOIDC_HOST siwx-oidc.example.com \
  --LETSENCRYPT_EMAIL you@example.com

# Update siwx-oidc to latest image
# IMPORTANT: Always restart Synapse after redeploying siwx-oidc.
# Synapse caches the OIDC JWK on startup; if siwx-oidc restarts with a
# different key derivation, Synapse will reject ID tokens (bad_signature).
docker compose pull siwx-oidc
docker compose up -d siwx-oidc
docker compose restart matrix_synapse

# Rebuild Synapse image after entrypoint changes
docker compose build matrix_synapse

# Apply a homeserver.yaml fix to a live deployment
docker compose exec matrix_synapse \
  yq -i '.oidc_providers[0].client_auth_method = "client_secret_post"' /data/homeserver.yaml
docker compose restart matrix_synapse
```

## Renaming a live homeserver.yaml key (existing deployments)

If `token_endpoint_auth_method` was written by an older entrypoint, rename it in place:

```bash
docker compose exec matrix_synapse sh -c '
  yq -i "del(.oidc_providers[0].token_endpoint_auth_method)" /data/homeserver.yaml &&
  yq -i ".oidc_providers[0].client_auth_method = \"client_secret_post\"" /data/homeserver.yaml
'
docker compose restart matrix_synapse
```
